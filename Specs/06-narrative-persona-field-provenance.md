# Spec 06: Narrative Persona Field Provenance

Priority: high
Status: ready

## Problem

The project wants Korea-style persona rows with rich narrative fields such as
professional persona, hobbies, travel, culinary preferences, skills, and career
goals.

ABS Census data can constrain structured demographic seed fields, but it does
not contain population distributions for most narrative persona fields. If those
fields are migrated from an external persona dataset or generated directly, the
project risks overstating statistical integrity by treating synthetic colour as
measured Australian population data.

The system needs a field-provenance contract before narrative generation is
expanded. This contract must distinguish:

- fields sampled from ABS distributions;
- fields conditioned by ABS-backed seed attributes;
- fields generated as narrative expansions;
- fields that should never be presented as statistically representative.

## Desired Behavior

The Australian persona pipeline must preserve statistical integrity by keeping
measured structured fields separate from generated narrative fields.

Every final persona field must declare one of these provenance classes:

| Provenance class | Meaning |
| --- | --- |
| `generated_identifier` | Deterministic or random ID, not a population fact. |
| `sampled_from_abs` | Directly sampled from an ABS-backed semantic or PGM distribution. |
| `conditioned_from_abs` | Derived or selected using ABS-backed seed fields, but not directly measured as the final field. |
| `generated_constrained` | Generated by Data Designer or an LLM using the fixed seed and validators. |
| `generated_unconstrained` | Generated synthetic colour with no statistical claim; avoid for v1 unless clearly labelled. |
| `reference_constant` | Controlled constant or reference value, such as `country = Australia`. |

Structured fields must preserve ABS-backed distributions and conditional
relationships where available:

```text
age, sex, marital_status, education_level, field_of_study, occupation,
income_band, family_type, housing_type, labour_force_status, ancestry,
country_of_birth, language_used_at_home, state, SA2, LGA
```

Narrative fields must be generated from the fixed structured seed and must not
contradict it:

```text
professional_persona
sports_persona
arts_persona
travel_persona
culinary_persona
family_persona
persona
cultural_background
skills_and_expertise
skills_and_expertise_list
hobbies_and_interests
hobbies_and_interests_list
career_goals_and_ambitions
```

The final dataset must be able to state:

```text
Structured demographics preserve ABS-derived distributions. Narrative persona
fields are synthetic expansions constrained by those demographics and local
context; they are not measured Census facts.
```

## Non-Goals

- Do not claim hobbies, skills, tastes, goals, or narrative personas are
  statistically representative of Australians unless a separate source and
  validation spec supports that claim.
- Do not reclassify Korean persona rows into Australian rows and treat the
  result as ABS-grounded.
- Do not use external persona rows as hidden population data.
- Do not generate final persona prose in this spec.
- Do not build PGM marts in this spec.
- Do not ingest non-ABS survey sources in this spec.

## Field Provenance Contract

The initial field map should be represented as configuration or a typed Python
constant before narrative generation is expanded.

Recommended first-pass map:

| Field | Provenance | Source or constraint |
| --- | --- | --- |
| `uuid` | `generated_identifier` | Generated per persona row. |
| `sa2_code` | `sampled_from_abs` | SA2 population/geography selection. |
| `sa2_name` | `sampled_from_abs` | ABS geography dimension. |
| `state` | `sampled_from_abs` | ABS geography dimension. |
| `country` | `reference_constant` | Usually `Australia`. |
| `age` | `conditioned_from_abs` | Exact age sampled inside an ABS `G04` age band. |
| `age_band` | `sampled_from_abs` | `G04`. |
| `sex` | `sampled_from_abs` | `G04`. |
| `marital_status` | `sampled_from_abs` | `G05`. |
| `family_type` | `conditioned_from_abs` | `G29`, with household/family grain caveat. |
| `household_relationship` | `sampled_from_abs` | `G27`. |
| `household_size` | `conditioned_from_abs` | `G35`; household-grain context, not a direct individual attribute. |
| `household_composition` | `conditioned_from_abs` | `G35`; household-grain context for plausibility and validation. |
| `housing_type` | `sampled_from_abs` | `G36`. |
| `tenure_landlord` | `conditioned_from_abs` | `G37`. |
| `education_level` | `sampled_from_abs` | `G16` and `G49`. |
| `field_of_study` | `sampled_from_abs` | `G50`; do not call this `bachelors_field` unless transformed. |
| `labour_force_status` | `sampled_from_abs` | `G46`. |
| `occupation` | `sampled_from_abs` | `G60`, for employed-person context. |
| `industry` | `conditioned_from_abs` | `G54`, often context for occupation/professional prose. |
| `income_band` | `sampled_from_abs` | `G17`. |
| `ancestry` | `sampled_from_abs` | `G08`. |
| `country_of_birth` | `sampled_from_abs` | `G09`. |
| `language_used_at_home` | `sampled_from_abs` | `G13`. |
| `english_proficiency` | `sampled_from_abs` | `G13`. |
| `indigenous_status` | `sampled_from_abs` | `G07`, with privacy/small-count restrictions. |
| `defence_service_status` | `conditioned_from_abs` | `G22`; not equivalent to Korean military status. |
| `rent_band` | `conditioned_from_abs` | Deferred `G40`; use only after rented-household context is sampled. |
| `accessibility_support_context` | `conditioned_from_abs` | Deferred `G18`; use only for reviewed support/accessibility use cases. |
| narrative persona fields | `generated_constrained` | Data Designer/LLM output constrained by the fixed seed. |

## Validator Requirements

Narrative generation must include validators that reject contradictions between
generated text and the structured seed.

Minimum validator rules:

- If `occupation` is `Retired` or `labour_force_status` is not employed, do not
  generate prose saying the person currently works full-time.
- If `age` is below adult scope, do not generate adult-only occupation, income,
  or family claims.
- If `country_of_birth = Australia`, do not describe the person as a recent
  migrant unless a separate arrival/migration field supports it.
- If `language_used_at_home = English`, do not imply the person mainly speaks
  another language at home.
- If `income_band` is low, do not generate routine luxury travel or spending
  habits as if they are ordinary.
- If `family_type` or `household_relationship` implies a lone-person household,
  do not generate text saying the person lives with spouse and children.
- If `housing_type` is an apartment, do not generate text that depends on a
  detached house unless the text is explicitly aspirational.
- If `indigenous_status` is present, do not generate cultural claims beyond the
  sampled field without a reviewed source and use case.

## Migration Guidance For Nemotron Personas Korea

Use the Korea dataset for:

- output schema shape;
- field naming inspiration;
- examples of narrative density and tone;
- validator ideas;
- prompt exemplars where demographic content is removed or adapted.

Do not use the Korea dataset for:

- Australian demographic distributions;
- hidden correlations between hobbies, housing, occupation, family, or culture;
- direct row rewriting into Australian Census buckets;
- evidence that narrative fields are representative of Australians.

Preferred migration path:

```text
Nemotron Personas Korea
  -> schema and style reference only

ABS Census semantic topics
  -> structured Australian seed

Data Designer
  -> constrained narrative expansion

validators and provenance tags
  -> final persona row with honest statistical claims
```

## Edge Cases

- Sparse SA2 partitions may require state-level or national fallback. Any
  fallback must be recorded in provenance or sampler diagnostics.
- Household/family tables are not person-level tables. They can condition
  plausibility but must not be treated as direct individual facts without an
  allocation rule.
- `Not stated`, `Not applicable`, totals, and `Persons` aggregates must not feed
  normal narrative generation as preferred values.
- Sensitive fields such as Indigenous status, religion, health, disability, and
  defence service need stricter use-case review before narrative prose mentions
  them.
- Generated list fields must be consistent with their paired text fields.

## Acceptance Criteria

- [ ] A machine-readable field provenance map exists.
- [ ] Every final persona column is classified into exactly one provenance
      class.
- [ ] Narrative fields are classified as `generated_constrained` or
      `generated_unconstrained`, not `sampled_from_abs`.
- [ ] Data Designer persona generation uses the provenance map or an equivalent
      contract when constructing prompts and validators.
- [ ] Validators reject generated text that contradicts structured seed fields.
- [ ] Documentation states which final fields preserve ABS-derived statistical
      claims and which are synthetic expansions.
- [ ] Relevant tests and lint checks pass or have documented unrelated blockers.

## Known Risks

- The generated narrative can sound more precise than the underlying data.
  Prompts and docs must avoid implying measured certainty.
- Adding external survey sources later may improve plausibility but will create
  new provenance classes and geographic-grain issues.
- Over-constraining generation may make persona prose bland; under-constraining
  it breaks integrity.
- Some useful Australian context, such as sports, arts, food, and travel, may
  need non-Census sources before it can carry any statistical claim.
